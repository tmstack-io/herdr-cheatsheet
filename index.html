Herdr チートシート

AIコーディングエージェント向けターミナルマルチプレクサ「Herdr」の、よく使うキー操作とCLIコマンドをまとめた非公式の日本語リファレンスです。クリックでコピーできます。

操作の基本： ctrl+b → 離す → c ＝ 新規タブを作成（prefix+c）
基本5つ ペイン タブ ワークスペース/セッション コピーモード CLI リモート Agent Skill 状態凡例

まず覚える5つ（prefixは既定で ctrl+b）

prefix+c新規タブ
prefix+v右にペイン分割
prefix+-下にペイン分割
prefix+h/j/k/lペイン間移動
prefix+qデタッチ（サーバーとagentは動き続ける）
全バインド一覧はいつでも prefix+? で確認できます（設定変更後も現在値を反映）。

ペイン操作

prefix+zフォーカス中ペインをズーム
prefix+xペインを閉じる
prefix+shift+h/j/k/lペイン入れ替え
prefix+rリサイズモード
prefix+[コピーモードへ

タブ操作

prefix+n / p次 / 前のタブ
prefix+1..9タブ番号へジャンプ
prefix+shift+tタブをリネーム
prefix+shift+xタブを閉じる

ワークスペース / セッション

prefix+shift+n新規ワークスペース
prefix+shift+wワークスペースをリネーム
prefix+shift+dワークスペースを閉じる
prefix+g検索可能なジャンプ先ピッカー
prefix+bサイドバー表示切替

コピーモード（prefix+[ で開始）

h/j/k/l, w/b/e, { }カーソル移動（単語・段落単位含む）
v / space選択開始
y / enterコピーして終了
q / escコピーモード終了

CLIコマンド抜粋

セッション / サーバー

herdr server stop
既定セッションを完全に停止（パネは全て終了）
herdr session list [--json] herdr session attach <name> herdr session stop <name> [--json] herdr session delete <name> [--json]
名前付きセッションの一覧・接続・停止・削除

ワークスペース / worktree

herdr workspace close <id>
Herdr内部の状態のみ閉じる（Git worktree自体は残る）
herdr worktree create --branch NAME [--base REF] herdr worktree open --path PATH herdr worktree remove --workspace ID [--force]
Git worktreeを作成・開く・削除しつつワークスペース化

ペイン制御（エージェント向け）

herdr pane split 1-2 --direction right --no-focus herdr pane run <pane_id> "npm run dev" herdr wait output <pane_id> --match "ready" --timeout 30000 herdr pane read <pane_id> --source recent --lines 20
分割 → 実行 → 出力待機 → 読み取り、の定番フロー

エージェント状態待機・統合

herdr wait agent-status 1-1 --status done --timeout 120000
隣のエージェントが done になるまで待つ
herdr integration install <agent>
対応: pi, codex, claude, copilot, devin, droid, kimi, opencode, kilo, hermes, qodercli, cursor など

リモート利用の2パターン

① SSHで入って起動（tmux方式）

ssh you@your-server herdr
Herdrはサーバー上で完結。シンプルで実用的。ローカルの画像クリップボード連携は不可。

② ローカルからシンクライアント接続

herdr --remote workbox herdr --remote ssh://you@server:2222
ローカルHerdrがSSH越しに接続。画像クリップボードペーストなどをブリッジ可能。

Agent Skill（エージェントにHerdrを操作させる）

npx skills add ogulcancelik/herdr --skill herdr -g
全対応エージェントにグローバルインストール（-g省略でプロジェクト限定）

⚠ ガードレール：環境変数 HERDR_ENV=1 が無い場合、エージェントは「Herdr管理下のpaneで動いていない」と表明して停止します。

エージェント状態の凡例

blocked（入力待ち） working（実行中） done（完了・未確認） idle（完了確認済み/待機） unknown（判別不能）
